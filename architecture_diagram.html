LLM Generation Layer Dual Model Routing
vLLM
Docker Container
Qwen3-8B (SLM)
Tasks:
- Email classification (Node 2)
- Sentiment analysis (Node 3)

Why SLM is enough:
- Structured output (JSON)
- Bounded categories
- Fast: ~0.3-0.8s/call
- Free (self-hosted GPU)
LiteLLM
Proxy Container
Unified Router
OpenAI-compatible API (:4000/v1)

Routing:
- Primary: vLLM (Qwen3-8B)
- Fallback: Anthropic API

Features:
- Load balancing
- Retry logic
- litellm_config.yaml
Anthropic API
External
Claude Sonnet 4.6 (LLM)
Tasks:
- Response generation (Node 6)
- Fallback for failed SLM calls

Why LLM is needed:
- Free-form customer-facing text
- Tone awareness (empathy, retention)
- Synthesizes all context
- Company reputation on the line
- ~2-4s/call, ~$0.02-0.04/email
